# Phase 1: Design Review

Design Review happens during Pre-Con, before construction starts. The QAR reviews specs and drawings to catch issues that would otherwise cause RFI chaos or rework in the field.

## What QAR Reviews

### 1. Spec Clarity
- Are requirements unambiguous?
- Are there conflicts between spec sections?
- Are referenced standards current?

### 2. Spec-Drawing Consistency
- Does the drawing match the spec?
- Are details missing that specs require?
- Do specs reference drawings that don't exist?

### 3. Constructability
- Can this actually be built as shown?
- Are there spatial conflicts (MEP clashes)?
- Are acessibility requirements met?

### 4. Completeness
- Have all trades been addressed?
- Are details adequate for construction?
- Are coordination drawings required?

---

## How to Document Findings

Design Review findings are typically submitted as a **Design Review Comment** or **Constructability Review**.

### Standard Format:

```
Project: [Project Name, Contract #]
Date: [Date]
Reviewer: [Name, Role]
Document Type: [Spec Section / Drawing / RFI]
Document #: [Spec #, Drawing #, etc.]

FINDING #: [1, 2, 3...]
CATEGORY: [Spec Conflict / Constructability / Missing Detail / Clarification]
DISCIPLINE: [Architectural / Structural / Mechanical / Electrical / Plumbing]

DESCRIPTION:
[What's wrong or missing]

SPEC/DRAWING REFERENCE:
[Cite the specific requirement]

RECOMMENDED RESOLUTION:
[Suggested fix]

PRIORITY: [Critical / Major / Minor]
```

---

## Common Findings by Trade

### Architectural
- Door schedules not matching door types in specs
- Finish materials specified not available
- ADA clearance issues

### Structural
- Load paths not clearly shown
- Connection details missing
- Steel moment frames not coordinated with architectural plans

### Mechanical (HVAC)
- Equipment clearances not shown
- Duct routing conflicts with structure
- Diffuser locations not coordinated with ceiling layout

### Electrical
- Panel locations not accessible
- Conduit routing conflicts
- Lighting controls not detailed

### Plumbing
- Pipe routing conflicts
- Fixture elevations not coordinated with floor heights
- Fire stopping details missing at penetrations

---

## Reference Documents Needed

To perform Design Review, QAR needs:
- Project Specifications (spec book)
- Architectural Drawings
- Structural Drawings
- MEP Drawings (Mechanical, Electrical, Plumbing)
- Applicable UFGS references
- Project Schedule (for coordination)

---

## Deliverables

Typical Design Review output:
- Design Review Comment Matrix
- Constructability Review Report
- RFI Log (issues requiring contractor RFI)
- Spec/Drawing Deficiency List
